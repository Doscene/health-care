# 02 角色化首页需求

## 1. 功能概述

首页根据用户角色呈现完全不同布局：老人端大字体简化版、年轻患者端数据深度版、子女端远程关怀版。三个版本共享底部 Tab，但首页内容和交互差异巨大。

---

## 2. 用户故事

| 角色 | 场景 | 期望结果 |
|------|------|----------|
| 老人 | 打开 App，只看今天该干啥 | 首页大字告诉我还有几件事没做，点一下就能记录 |
| 年轻患者 | 上班前快速看一眼血压趋势 | 首页展示本周血压趋势图表，今天待办，数据洞察 |
| 子女 | 上班路上看看爸妈情况 | 首页展示父母健康卡片，有无异常一眼看清 |

---

## 3. 老人首页

### 3.1 整体风格
- **全局大字体**：正文 ≥ 18px，标题 ≥ 24px
- **高对比度**：白底深灰字，关键按钮绿色
- **减少信息密度**：一屏仅 3-4 个模块
- 昵称显示：顶部大字"早上好，张建国"

### 3.2 模块一：今日待办
- 标题：**"今天还剩 X 件事"**
- 任务卡片列表，每项包含：
  - 图标（💊服药 / 🩺测血压 / 🍬测血糖）
  - 任务名称 + 时间说明（如"早餐后 1 片 · 8:00"）
  - 点击卡片 → 勾选完成（打勾 + 划线变灰）
- 已完成任务显示为灰色 + 删除线 + ✅
- 任务来源：用药计划 + 测量计划

### 3.3 模块二：语音记录大按钮
- 全宽绿色大按钮，呼吸动画光环
- 内容：
  - 麦克风图标
  - **"按住说话，快速记录"**
  - 副文字："血压 135 85" 或 "吃了降压药"
- 点击 → 跳转快速记录页（语音界面）

### 3.4 模块三：最近的数字
- 标题："最近的数字" + "本周平均"
- 2 列网格卡片：
  - 蓝色卡片：收缩压/舒张压（大字 132/84）
  - 黄色卡片：空腹血糖（大字 6.8）
- 每张卡片标注单位 + 正常/偏高判断

### 3.5 顶部
- 头像（右侧圆形）+ 问候语"早上好，张建国"
- 问候语按时间段变化：早上好 / 下午好 / 晚上好

## 4. 年轻患者首页

### 4.1 整体风格
- 标准字体，深色背景数据图表
- 信息密度中等，注重数据可视化和洞察
- 强调"掌控感"，非"病患感"

### 4.2 模块一：顶部 + 人生阶段标签
- 昵称："早安，李想" + 头像
- **人生阶段标签**（多标签可并存）：
  - 🍼 备孕模式（粉色胶囊）
  - 💼 职场模式（灰色胶囊）
  - 点击标签 → 进入对应专区

### 4.3 模块二：今日待办
- 标题："今日待办"
- 任务卡片（紧凑版）：
  - 药物名 + 安全标注（如"孕期安全"）
  - 时间 + 目标值
  - 操作按钮："确认" / "记录"

### 4.4 模块三：本周趋势图表
- 标题："本周趋势" + "较上周 -8mmHg"
- 深色背景卡片，包含：
  - 平均收缩压大数字 + 变化箭头
  - 7 天柱状图（周一至周日）
  - X 轴：周一到周日标签

### 4.5 模块四：快捷记录入口
- 4 宫格：血压 | 血糖 | 饮食 | 应酬
- 每格 = 图标 + 文字
- 点击 → 跳转对应记录表单

### 4.6 模块五：数据洞察卡片
- 浅紫色背景
- 灯泡图标 + 智能分析文字
- 例："本周三次应酬后次日晨起血压平均升高 15mmHg，试试减少啤酒摄入？"
- **MVP 阶段**：基于规则模板生成（非 AI）

## 5. 子女首页

### 5.1 整体风格
- 标准字体，卡片式布局
- 关注点：父母状态概览 + 异常提醒
- 每周一看的频率设计

### 5.2 模块一：顶部
- "你好，张小红" + 头像
- 副文字："父母在老家，每周看一眼，心里踏实"

### 5.3 模块二：父母健康卡片
- 每人一张卡片，包含：
  - 头像 + 基本信息（姓名/病种/年龄）
  - 状态标签：🟢 本周平稳 / 🟡 血糖偏高 / 🔴 需关注
  - 关键指标 3 列：
    - 平均收缩压/空腹血糖数值
    - 服药依从性百分比
    - 漏测次数 / 异常天数
  - 如有异常 → 黄色/红色提示条："建议电话关心一下饮食情况"

### 5.4 模块三：周报入口
- 深色背景大按钮："查看上周健康简报"
- 点击 → 跳转周报页

## 7. 数据结构要点

```typescript
// 首页聚合数据（各角色不同）
HomeData {
  userId: string
  greeting: string              // "早上好" / "下午好" / "晚上好"
  pendingTasks: Task[]          // 今日待完成
  completedTasks: Task[]        // 今日已完成
  recentMetrics: {              // 最近健康指标
    avgSystolic: number
    avgDiastolic: number
    avgFastingGlucose: number
    periodLabel: string         // "本周平均"
  }
  weeklyTrend: {                // 年轻患者专属
    avgSbp: number
    change: number
    dailyData: number[]         // 7天数据
  }
  insight: string               // 数据洞察文字
  parentsOverview?: ParentCard[] // 子女专属
  lifeStageTags: string[]       // 年轻患者专属：['pregnancy','career']
}

ParentCard {
  userId: string
  name: string
  disease: string
  status: 'normal' | 'attention' | 'warning'
  avgSbp?: number               // 高血压
  avgGlucose?: number           // 糖尿病
  adherence: number             // 依从性百分比
  missedCount: number
  abnormalDays: number
  suggestion?: string
}
```

## 8. 接口需求概要

| 接口 | 方法 | 路径 | 说明 |
|------|------|------|------|
| 获取首页数据 | GET | /api/home | 根据角色返回不同结构 |
| 完成任务 | POST | /api/tasks/:id/complete | |
| 获取周趋势数据 | GET | /api/metrics/weekly-trend | userId + metricType |

---

## 9. 异常处理

| 场景 | 处理 |
|------|------|
| 首页数据加载失败 | 展示骨架屏，点击重试 |
| 无待办任务 | 显示"今天已全部完成 🎉" |
| 无近期数据 | 显示引导卡片"开始第一次记录" |
| 无父母数据 | 子女端引导"邀请父母加入家庭圈" |
