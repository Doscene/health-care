# 03 智能用药管理需求

## 1. 功能概述

拍药盒识别药品 → 生成服药计划 → 到点提醒 → 一键确认 → 多药冲突检测 → 服药依从性追踪。核心设计原则：**老人不录入药名，只拍盒子**。

---

## 2. 用户故事

| 角色 | 场景 | 期望结果 |
|------|------|----------|
| 子女 | 帮父母设置用药，不知道父母具体吃什么药 | 拍父母药盒，自动识别药品信息，一键生成服药计划 |
| 老人 | 到点吃药，但记不住药名和剂量 | 手机弹出提醒"该吃降压药了"，点一下就确认吃了 |
| 年轻患者 | 备孕期间担心药物安全性 | 用药表标注孕期安全等级，提示需要替换的药物 |

---

## 3. 药品识别

### 3.1 拍药盒识别（OCR）
- **MVP 范围**：支持识别药品名称、规格（如"苯磺酸氨氯地平片 5mg"）
- **流程**：
  1. 点击拍照按钮 → 调用相机
  2. 拍药盒正面（含药品名+规格）
  3. 上传图片 → OCR 识别
  4. 返回识别结果，用户确认/修改
  5. 如识别失败 → 手动输入

### 3.2 手动录入
- 药品名称（必填）
- 规格（如 5mg/片）
- 用法用量：每次 X 片，每日 Y 次
- 服用时间：早餐前 / 早餐后 / 午餐后 / 晚餐后 / 睡前
- 起止日期（可选）
- 备注（可选）

---

## 4. 服药计划

### 4.1 自动生成
- 根据药品信息和服用频次，自动生成每日服药计划
- 示例：
  ```
  苯磺酸氨氯地平片 5mg | 每日1次 | 早餐后 | 8:00
  二甲双胍 0.5g        | 每日2次 | 早晚餐后 | 8:00 / 19:00
  ```

### 4.2 管理功能
- 查看所有药品列表
- 编辑药品信息
- 暂停/恢复某药品
- 删除药品（需二次确认）
- 服药计划日历视图

---

## 5. 用药提醒

### 5.1 提醒方式
- 手机推送通知（主方式）
- 提醒内容：药品名称 + "该吃药了" + 当前时间
- 老人端：大字号通知 + 震动

### 5.2 确认方式
- **点击确认**：通知栏直接操作，标记"已服用"
- **语音确认**（扩展）：老人说"吃了"即可确认
- **漏服处理**：
  - 提醒后 30 分钟内未确认 → 视为漏服
  - 连续漏服 2 次 → 触发询问 + 通知子女

### 5.3 提醒策略
- 到点提醒 → 5 分钟后未确认 → 再提醒一次
- 连续 2 次漏服 → 询问"是否身体不适？"或"是否忘记？"
- 连续漏服 ≥ 3 次 → 通知紧急联系人

## 6. 多药冲突检测

### 6.1 冲突规则（MVP内置规则表）
| 药物组合 | 风险提示 |
|----------|----------|
| ACEI 类降压药 + 二甲双胍 | 建议定期监测肾功能 |
| 噻嗪类利尿剂 + 胰岛素 | 注意血糖波动 |
| β受体阻滞剂 + 胰岛素 | 可能掩盖低血糖症状 |
| 华法林 + 阿司匹林 | 出血风险增加 |

### 6.2 检测时机
- 新增药品时实时检测
- 已有用药组合定期（每周）重新评估

### 6.3 冲突呈现
- 黄色警告条："检测到潜在药物相互作用，建议咨询医生"
- 不阻断用药，仅做提示

## 7. 服药日历与依从性

### 7.1 服药日历
- 月视图日历
- 每天用颜色标记：🟢 全部按时 / 🟡 有延迟 / 🔴 有漏服

### 7.2 依从性统计
- 当前月 / 累计：
  - 按时服药率（百分比）
  - 总漏服次数
  - 平均延迟时间
- 在首页和报告中使用

---

## 8. 页面/交互说明

### 8.1 用药表列表页（"我的"Tab 内进入）
- 药品卡片列表：药品名 + 规格 + 时间 + 状态（进行中/已暂停）
- 顶部 "+" 添加药品
- 左滑药品卡片 → 编辑 / 暂停 / 删除

### 8.2 添加药品页
- 顶部两个 Tab：拍照识别 | 手动输入
- **拍照识别**：相机取景框 → 拍照 → 识别中 loading → 确认结果
- **手动输入**：表单字段（药品名/规格/用法/时间/备注）

### 8.3 服药提醒弹窗
- 系统通知样式（不进入 App 即可操作）
- 内容：药品名称 + 剂量 + "该吃药了"
- 两个操作按钮："已服用" / "稍后提醒"

## 9. 数据结构要点

```typescript
Medication {
  id: string
  userId: string
  name: string               // 药品名称
  specification: string      // 规格，如"5mg/片"
  dosagePerTime: number      // 每次用量（片数）
  frequencyPerDay: number    // 每日次数
  timeSlots: string[]        // 服药时段：["breakfast_after","dinner_after"]
  remindTimes: string[]      // 提醒时间：["08:00","19:00"]
  startDate: date
  endDate?: date
  status: 'active' | 'paused' | 'stopped'
  notes?: string
}

MedicationRecord {
  id: string
  medicationId: string
  userId: string
  scheduledTime: datetime
  actualTime?: datetime
  status: 'taken' | 'missed' | 'delayed' | 'skipped'
  confirmedBy: 'click' | 'voice'
}

DrugConflict {
  id: string
  medicationIds: string[]
  riskLevel: 'low' | 'medium' | 'high'
  description: string
  suggestion: string
}
```

## 11. 异常处理

| 场景 | 处理 |
|------|------|
| OCR 识别失败 | 提示"未能识别药品，请手动输入"，保留照片 |
| OCR 识别结果不确定 | 展示 Top 3 候选，让用户选择 |
| 提醒通知被系统屏蔽 | 在 App 内提示"请开启通知权限以确保按时提醒" |
| 已暂停药品到期 | 不提醒，但保留记录 |
| 网络异常时确认服药 | 本地暂存，联网后同步 |
