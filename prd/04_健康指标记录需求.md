# 04 健康指标记录需求

## 1. 功能概述

支持血压（收缩压/舒张压/心率）、血糖（空腹/餐前/餐后2h）的快速记录。核心设计原则：**一次记录不超过 10 秒**，支持语音输入和拍照识别。

---

## 2. 用户故事

| 角色 | 场景 | 期望结果 |
|------|------|----------|
| 老人 | 测完血压想说句话就记录好 | 按住说话"血压135 85"，自动识别并保存 |
| 年轻患者 | 出差在外，快速记录餐后血糖 | 10秒内完成选择类型+输入数值+保存 |
| 子女 | 帮父母代录血压数据 | 手动表单填入三项数值，保存后同步到家庭看板 |

---

## 3. 血压记录

### 3.1 记录字段
| 字段 | 类型 | 范围 | 单位 |
|------|------|------|------|
| 收缩压 | number | 60-280 | mmHg |
| 舒张压 | number | 30-160 | mmHg |
| 心率 | number (选填) | 40-200 | bpm |
| 记录时间 | datetime | 默认当前，可修改 | |

### 3.2 输入方式
- **手动输入**：三个数字输入框，大字体居中
- **语音输入**（主推）：按住说话"高压135低压85心率72" → 自动拆解填入
- **拍照识别**（基础OCR）：拍血压计屏幕 → 识别三个数字并填入表单

### 3.3 数值校验
- 收缩压 < 舒张压 → 提示"收缩压应大于舒张压，请检查"
- 极端异常值（如收缩压 280）→ 提示"数值异常，请确认是否正确"
- 心率不在 40-200 范围 → 提示"请输入正确心率"

---

## 4. 血糖记录

### 4.1 记录字段
| 字段 | 类型 | 范围 | 单位 |
|------|------|------|------|
| 测量类型 | enum | 空腹 / 餐前 / 餐后2h / 睡前 / 随机 | |
| 血糖值 | number | 1.0-35.0 | mmol/L |
| 记录时间 | datetime | 默认当前 | |

### 4.2 输入方式
- **手动输入**：先选类型（分段按钮），再输数值
- **语音输入**：按住说话"空腹血糖6.8" → 自动识别类型+数值
- 默认类型为"空腹"

### 4.3 目标提示
- 血糖保存后，根据类型和用户病种显示目标判断：
  - 空腹正常：3.9-6.1 mmol/L
  - 餐后2h正常：< 7.8 mmol/L
  - 提示："很好，在目标范围内" / "偏高，建议复测"

---

## 5. 语音快速记录

### 5.1 核心交互
1. 首页点击语音大按钮或记录页点击麦克风
2. 按住说话，松手结束
3. 语音上传 → 服务器 ASR 识别
4. 解析为结构化数据：
   - "血压135 85" → 收缩压135 + 舒张压85
   - "空腹血糖7.2" → 血糖类型空腹 + 数值7.2
   - "吃了降压药" → 触发用药确认
5. 展示解析结果 → 用户确认保存

### 5.2 语音解析规则
| 语音示例 | 解析结果 |
|----------|----------|
| "血压 135 85" | 收缩压135, 舒张压85 |
| "高压135低压85心率72" | 收缩压135, 舒张压85, 心率72 |
| "空腹血糖 6.8" | 血糖类型空腹, 数值6.8 |
| "餐后两小时 9.0" | 血糖类型餐后2h, 数值9.0 |
| "吃了降压药" | 触发用药确认 |

### 5.3 解析失败处理
- 提示"未能识别，请手动记录"
- 保留录音文本，支持手动修改

## 6. 拍照识别（血压计屏幕）
- 拍照界面含取景辅助框（引导对准屏幕）
- OCR 识别屏幕上的三个数值区域
- 返回识别结果填入表单
- 识别失败回退手动输入

---

## 7. 记录列表

### 7.1 血压记录列表
- 按时间倒序
- 每行：时间 + 收缩压/舒张压 + 心率 + 状态颜色标签
- 状态颜色：🟢 正常 / 🟡 偏高 / 🔴 危险
- 点击 → 查看详情 / 编辑 / 删除

### 7.2 血糖记录列表
- 按时间倒序
- 每行：时间 + 测量类型标签 + 血糖值 + 状态颜色标签
- 点击 → 详情/编辑/删除

### 7.3 筛选
- 按日期范围筛选
- 按类型筛选（仅血糖）
- 按状态筛选（正常/偏高/危险）

---

## 8. 页面/交互说明

### 8.1 快速记录页（主入口）
- 顶部标题"快速记录" + 返回按钮
- 记录类型选择区（2x2 网格）：
  - 🩺 血压（收缩/舒张/心率）
  - 🍬 血糖（空腹/餐前/餐后）
  - 💊 用药（拍照或手动确认）
  - 🍱 饮食（拍照/语音记录）
- 语音输入演示区（波形动画 + 麦克风按钮）
- 点击类型 → 展示对应表单
- 保存成功 → 弹出成功提示 → 返回首页

### 8.2 血压表单
- 3 列数字输入框（收缩压/舒张压/心率）
- 每个输入框：顶部标签 + 大数字输入框
- 绿色保存按钮

### 8.3 血糖表单
- 顶部类型选择器（分段按钮：空腹/餐前/餐后2h）
- 血糖值数字输入框（支持一位小数）
- 绿色保存按钮

### 8.4 语音记录界面
- 中央大圆形麦克风按钮（按住说话）
- 波形动画（5条跳动竖线）
- 提示文字："点击模拟：空腹血糖 7.2"
- 松手后识别的文字展示

### 8.5 记录成功弹窗
- 半透明遮罩 + 居中白色弹窗
- ✅ 图标 + "记录成功"
- 副文字"数据已同步至家庭看板"
- "知道了"按钮关闭弹窗

## 9. 数据结构要点

```typescript
BloodPressureRecord {
  id: string
  userId: string
  systolic: number           // 收缩压
  diastolic: number          // 舒张压
  heartRate?: number         // 心率（选填）
  recordedAt: datetime
  inputMethod: 'manual' | 'voice' | 'ocr'
  source?: string            // 语音原文或OCR图片URL
}

BloodSugarRecord {
  id: string
  userId: string
  type: 'fasting' | 'before_meal' | 'after_meal_2h' | 'bedtime' | 'random'
  value: number
  recordedAt: datetime
  inputMethod: 'manual' | 'voice'
  source?: string
}
```

## 10. 接口需求概要

| 接口 | 方法 | 路径 | 说明 |
|------|------|------|------|
| 语音识别 | POST | /api/speech/recognize | body: { audio } |
| 保存血压记录 | POST | /api/records/blood-pressure | body: BloodPressureRecord |
| 保存血糖记录 | POST | /api/records/blood-sugar | body: BloodSugarRecord |
| 血压记录列表 | GET | /api/records/blood-pressure | query: userId, startDate, endDate |
| 血糖记录列表 | GET | /api/records/blood-sugar | query: userId, startDate, endDate, type |
| OCR 血压计 | POST | /api/ocr/bp-monitor | body: { image } |
| 编辑记录 | PUT | /api/records/:type/:id | |
| 删除记录 | DELETE | /api/records/:type/:id | |

---

## 11. 异常处理

| 场景 | 处理 |
|------|------|
| 语音权限未授权 | 提示"请在设置中开启麦克风权限" |
| 语音识别超时 | 提示"识别超时，请重试" |
| 语音内容无法解析 | 展示识别文本，引导手动修改 |
| 数值超出合理范围 | 红色提示"数值异常，请确认" |
| 保存失败 | Toast + 本地暂存，联网后重试 |
