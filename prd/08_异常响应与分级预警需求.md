# 08 异常响应与分级预警需求

## 1. 功能概述

四级风险响应体系（绿/黄/橙/红），从自动记录到一键呼救的渐进式响应。核心原则：**分级响应，避免骚扰；用"关心"而非"警告"的语气**。

---

## 2. 用户故事

| 角色 | 场景 | 期望结果 |
|------|------|----------|
| 老人 | 血压突然很高但自己没感觉 | App 立即提醒本人 + 通知子女，避免危险 |
| 子女 | 爸爸血压连续偏高，想知道要不要紧 | App 温和告知"建议电话关心一下"，而非吓人的警报 |
| 年轻患者 | 工作应酬后血压偏高 | 系统关联分析应酬与血压关系，给出数据洞察而非警告 |
| 退休夫妻 | 一方漏服药多次 | 提醒配偶关注，询问原因 |

## 3. 四级风险体系

### 绿色 🟢 —— 正常
| 项目 | 条件 | 响应 |
|------|------|------|
| 触发 | 数据在目标范围内 | 自动记录，无额外动作 |
| 血压 | 收缩压 < 140 且 舒张压 < 90 | |
| 血糖 | 空腹 3.9-7.0，餐后 < 10.0 | |

### 黄色 🟡 —— 单次异常
| 项目 | 条件 | 响应 |
|------|------|------|
| 触发 | 单项指标异常 | 推送健康科普 + 建议复测 |
| 血压 | 收缩压 140-159 或 舒张压 90-99 | 提示"血压偏高，建议休息后复测" |
| 血糖 | 空腹 7.0-8.0 或 餐后 10.0-13.0 | 提示"血糖偏高，建议记录饮食" |
| 漏服 | 单次漏服 | 温和提醒补服或确认 |

### 橙色 🟠 —— 持续异常
| 项目 | 条件 | 响应 |
|------|------|------|
| 触发 | 连续3天偏离目标 或 症状组合 | 通知紧急联系人 + 建议就医 |
| 血压 | 连续3天收缩压 > 150 | "近3天血压偏高，建议联系家庭医生" |
| 血糖 | 连续3天空腹 > 8.0 | "血糖持续偏高，建议复诊调整方案" |
| 漏服 | 连续漏服 ≥ 2 次 | 询问原因 → 子女收到通知 |
| 症状组合 | 血压高 + 头晕/胸闷 | 立即触发橙色 |

### 红色 🔴 —— 危险值
| 项目 | 条件 | 响应 |
|------|------|------|
| 触发 | 疑似危险值 | 立即提醒患者 + 紧急联系人 + 一键呼救 |
| 血压 | 收缩压 ≥ 180 或 舒张压 ≥ 110 | "血压危险值！请立即休息并联系医生" |
| 血糖 | 空腹 ≥ 13.9 或 ≤ 2.8 | "血糖危险值！请立即采取措施" |
| 低血糖 | 任何低血糖症状 + 血糖 ≤ 3.9 | "疑似低血糖，请立即摄入糖分" |
| 心梗疑似 | 剧烈胸痛 + 大汗 | 触发急救指南 + 一键拨120 |

## 4. 异常响应流程

### 4.1 橙色响应流程
```
连续3天血压 > 150
  └→ 系统弹出询问卡片："最近睡眠怎么样？盐是不是吃多了？"
       ├→ 患者回复"还好" → 记录 + 继续观察
       ├→ 患者回复"不舒服" → 建议就医 + 通知紧急联系人
       └→ 患者24h未回应 → 通知紧急联系人："爸爸近3天血压偏高，建议电话关心"
```

### 4.2 红色响应流程
```
血压 ≥ 180/110 录入
  └→ 立即全屏红色弹窗：
       "血压危险值 180/110！请立即休息"
       ├→ 按钮1：拨打120（一键呼救）
       ├→ 按钮2：联系紧急联系人
       └→ 按钮3：我知道了（确认后仍记录）
  └→ 同步推送通知给所有紧急联系人
       "【紧急】爸爸血压达到危险值 180/110，请立即联系"
```

## 5. 紧急联系人

### 5.1 联系人设置
- 位置："我的" → 紧急联系人
- 添加方式：从通讯录选择 或 输入手机号
- 最多设置 **5 人**
- 排序：可拖拽调整优先级（第一个为第一联系人）

### 5.2 通知策略
| 级别 | 通知谁 |
|------|--------|
| 黄色 | 仅患者本人 |
| 橙色 | 患者 + 第一紧急联系人 |
| 红色 | 患者 + 全部紧急联系人 + 推送120呼救建议 |

### 5.3 互为紧急联系人
- 夫妻可以在家庭页一键设置"互为紧急联系人"
- 一方异常数据另一方优先收到

---

## 6. 一键呼救

### 6.1 触发方式
- 红色风险弹窗内"拨打120"按钮
- "我的" → 紧急联系人 → 快捷呼救按钮
- 可作为桌面小组件（后续版本）

### 6.2 呼救信息自动准备
- 拨通120后，自动展示急救信息卡片：
  - 患者姓名、年龄
  - 已知疾病
  - 当前定位地址
  - 最近一次血压/血糖值
  - 紧急联系人电话
  - 过敏药物（如有录入）

---

## 7. 家庭急救包管理

### 7.1 急救包信息
- 常备药品列表 + 有效期
- 急救用品有效期（如硝酸甘油）
- 到期前 30 天提醒更换

### 7.2 急救情景指南
- 内置急救图文指南：
  - "胸痛怎么办？"
  - "低血糖怎么办？"
  - "打120时说什么？"
  - "硝酸甘油怎么舌下含服？"

---

## 8. 异常问询卡片

### 8.1 橙色级别问询
当触发橙色时，弹出温和问询卡片（非警告弹窗）：
```
血压最近波动有点大 📈
最近3天晨起血压偏高（152/96）
可能和什么有关？
  ○ 睡眠不好
  ○ 吃咸了
  ○ 忘记吃药
  ○ 最近压力大
  ○ 不知道 / 其他
[提交]
```

### 8.2 漏服问询
```
您连续2次没按时吃降压药了
是遇到什么情况吗？
  ○ 忘了
  ○ 身体不舒服
  ○ 药吃完了
  ○ 自己减量了
  ○ 其他
[提交]
```
- 如果是"自己减量" → 提示"建议遵医嘱用药，如需调整请咨询医生"
- 如果是"药吃完了" → 提醒复诊/购药

## 9. 页面/交互说明

### 9.1 异常弹窗（红色）
- 全屏半透明遮罩
- 底部白色卡片上滑
- ⚠️ 大图标 + 标题"血压危险值"
- 具体数值 + 说明文字
- 两个操作按钮："给紧急联系人打电话" / "我知道了"
- 不点"知道了"无法关闭（强制关注）

### 9.2 温和问询卡片（橙色/黄色）
- 原位卡片展示（非弹窗）
- 单选选项
- 提交后记录回答 + 更新风险状态

## 10. 数据结构要点

```typescript
RiskAlert {
  id: string
  userId: string
  level: 'green' | 'yellow' | 'orange' | 'red'
  triggerType: 'bp_high' | 'bg_high' | 'bg_low' | 'missed_dose' | 'symptom_combo'
  triggerValue: string
  status: 'active' | 'acknowledged' | 'resolved'
  notifiedContacts: string[]
  createdAt: datetime
  resolvedAt?: datetime
}

EmergencyContact {
  id: string
  userId: string
  name: string
  phone: string
  relation: string
  priority: number
  isMutual: boolean            // 是否互为紧急联系人
}

EmergencyKit {
  id: string
  familyId?: string
  items: EmergencyKitItem[]
}

EmergencyKitItem {
  name: string
  type: 'medicine' | 'supply'
  expireDate: date
  notes?: string
}

RiskInquiry {
  id: string
  alertId: string
  question: string
  options: string[]
  answer?: string
}
```

## 11. 接口需求概要

## 12. 异常处理

| 场景 | 处理 |
|------|------|
| 紧急联系人电话无法接通 | 依次拨打下一个优先级的联系人 |
| 用户拒绝通知权限 | 橙色/红色仍展示 App 内弹窗 |
| 告警确认超时（红色30分钟未确认） | 自动通知紧急联系人 |
| 急救包药品过期未更新 | 每次打开急救包页检查并提醒 |
