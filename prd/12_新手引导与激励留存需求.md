# 12 新手引导与激励留存需求

## 1. 功能概述

分角色新手引导（Onboarding）、成就徽章体系、连续打卡激励、家庭挑战、照护者支持、冷启动与长期留存策略。目标是让用户第一周完成核心行为，形成使用习惯。

---

## 2. 用户故事

| 角色 | 场景 | 期望结果 |
|------|------|----------|
| 子女 | 刚下载 App，不知道怎么帮父母设置 | 引导流程清晰：创建家庭→邀请父母→拍药盒设置用药 |
| 老人 | 子女帮忙弄好了，自己每天用 | 打开就看得到今天该干什么，不用想 |
| 年轻患者 | 想坚持记录但怕自己没毅力 | 连续记录解锁成就徽章，数据可视化增强掌控感 |
| 退休夫妻 | 一起用 App 互相监督 | 家庭目标完成解锁"默契家庭"徽章 |

---

## 3. 分角色 Onboarding

### 3.1 子女 Onboarding 流程
```
Step 1: 填写姓名
Step 2: 选择角色"我是家人（照护者）"
Step 3: 创建家庭健康圈 → 输入家庭名称
Step 4: 邀请父母（展示邀请码，引导分享）
Step 5: 代设置用药计划（引导拍照/OCR识别）
Step 6: 首次记录完成 → 解锁"家庭健康档案"电子封面
```

### 3.2 年轻患者 Onboarding 流程
```
Step 1: 填写姓名 + 出生年份
Step 2: 选择病种（高血压/糖尿病/双病）
Step 3: 完成疾病认知测试（5道简单题，破除常见误解）
      例："得了糖尿病不等于什么都不能吃 ✅"
Step 4: 设置个人目标（如"3个月内糖化血红蛋白降到7%以下"）
Step 5: 邀请配偶加入家庭圈
Step 6: 选择人生阶段（备孕/职场/日常）
Step 7: 首次完成血压/血糖记录 → 解锁"健康合伙人"身份
```

### 3.3 老人 Onboarding 流程
```
Step 1: 输入子女给的邀请码 或 扫描二维码
Step 2: 确认加入家庭圈（显示家庭名称和成员）
Step 3: 引导"试试按住说话记录"（语音教学）
Step 4: 完成首次记录 → 鼓励性反馈
```
- 如果无邀请码，跳过 Step 1-2，进入极简首页

### 3.4 Onboarding 设计原则
- 每步一个核心操作，不过多信息
- 可跳过，但标记"稍后在设置中完成"
- 老人端步骤 ≤ 3 步

## 4. 成就徽章体系

### 4.1 徽章列表

| 徽章名称 | 获取条件 | 类型 |
|----------|---------|------|
| 🌱 初来乍到 | 完成首次记录 | 一次性 |
| ⭐ 自律之星 | 连续 30 天按时服药 | 里程碑 |
| 📅 记录达人 | 累计记录 100 次 | 里程碑 |
| 🩺 血压达人 | 血压连续 7 天在目标范围内 | 里程碑 |
| 🍬 控糖能手 | 空腹血糖连续 7 天达标 | 里程碑 |
| 🤝 健康合伙人 | 夫妻共同完成月度目标 | 协作 |
| 👨‍👩‍👧 默契家庭 | 全家成员全部完成周任务 | 协作 |
| 🏃 运动之星 | 家庭周运动目标完成 | 协作 |
| 🎖️ 百日坚持 | 连续记录 100 天 | 终极 |
| 📖 学习达人 | 阅读 30 篇健康知识文章 | 行为 |

### 4.2 徽章展示
- "我的"页面顶部"我的成就"区域
- 已获得：彩色图标 + 名称 + 获得日期
- 未获得：灰色图标 + 名称 + 进度条 + 还差多少
- 获得时弹出庆祝动画

### 4.3 设计原则
- **正向激励**：庆祝做到的事，不惩罚没做到的
- **非竞争性**：不设排行榜，仅自己可见
- **轻量级**：不强制、不压迫

## 5. 连续打卡激励

### 5.1 连续记录
- 每天完成至少 1 项核心操作（记录血压/血糖/服药）
- 连续天数在首页展示
- 连续 7 天 → 解锁"健康习惯勋章"
- 连续 30 天 → 解锁"自律之星"

### 5.2 中断处理
- 中断不惩罚，温和鼓励"没关系，今天重新开始"
- 连续天数重置，但已获得的徽章永久保留
- 不展示"你断了几天"的负面信息

## 6. 家庭挑战（协作激励）

### 6.1 挑战类型
| 挑战 | 条件 | 奖励 |
|------|------|------|
| 周末健走 | 周末两天全家合计走 10 公里 | 运动之星徽章 |
| 血压稳定周 | 一周内血压全部在目标范围 | 血压达人徽章 |
| 全家全勤 | 全部成员一周内全部任务完成 | 默契家庭徽章 |

### 6.2 挑战进度
- 家庭页展示活跃挑战
- 进度条 + 参与成员头像 + 还差多少
- 挑战达成 → 全屏庆祝动画

## 7. 固定触点（留存策略）

### 7.1 每周触点
- **每周一早上 8:00**：推送"上周家庭健康简报"（子女端）
- **每周日晚上**：推送"本周总结"（患者端）

### 7.2 每月触点
- **每月 1 日**：推送"家庭健康月报"
- **季度末**：推送"季度故事报告"（年轻患者）

### 7.3 异常触点
- 橙色/红色预警时推送（关心的语气）

### 7.4 推送语气规范
| 场景 | ✅ 推荐 | ❌ 避免 |
|------|---------|---------|
| 漏药提醒 | "今天还没服药哦，现在补上吧" | "您又忘记吃药了" |
| 异常通知 | "妈妈近3天血压偏高，建议关心一下" | "警报：妈妈血压异常" |
| 鼓励 | "您连续 30 天按时服药，血压控制很棒！" | "本次血压偏高，请注意" |

## 8. 照护者支持（后续迭代基础）

### 8.1 照护者疲惫指数
- 位置："我的"页面（有照护角色时展示）
- 定期推送简短评估问卷（每月1次）：
  - "最近照顾家人感到累吗？1-5分"
  - "是否需要帮助资源？"
- 得分 ≥ 4 分 → 推荐喘息服务资源

### 8.2 照护技巧微课（后续迭代）
- 5 分钟短视频：
  - "如何帮助卧床老人翻身"
  - "糖尿病足的日常检查"
  - "如何跟父母沟通健康问题不吵架"

### 8.3 喘息服务资源
- 当照护者疲惫指数过高时
- 推荐本地上门护理、短期托老机构（对接第三方平台）

## 9. 冷启动策略

### 9.1 第一周（用户激活）
| 角色 | 策略 |
|------|------|
| 子女 | 帮父母设置好 → 首次记录完成 → 送"家庭健康档案"电子封面 |
| 年轻患者 | 疾病认知测试 → 设目标 → 解锁"健康合伙人"身份 |
| 老人 | 语音记录成功 → 正面鼓励反馈 → 降低使用门槛 |

### 9.2 第一个月（习惯养成）
| 策略 | 说明 |
|------|------|
| 连续 7 天记录 → 解锁"健康习惯勋章" | 行为锚定 |
| 夫妻组队完成周目标 → 解锁双人皮肤 | 社交激励 |
| 首次生成《季度故事报告》→ 支持分享 | 社交传播 |

### 9.3 持续留存
| 策略 | 说明 |
|------|------|
| 每周一早 8 点推送周报 | 固定触点 |
| 每月推送并发症风险变化 | 持续价值感 |
| 人生阶段提醒（备孕模式到期提醒等） | 生命周期 |

## 10. 页面/交互说明

### 10.1 Onboarding 流程
- 全屏引导页，每步一个核心操作
- 底部"下一步"按钮 + "跳过"链接
- 进度条（如 Step 2/5）
- 老人端无 Onboarding，直接进入首页

### 10.2 成就徽章展示
- "我的"页 → "我的成就"
- 徽章网格（4 列）
- 获得 → 彩色；未获得 → 灰色 + 进度文字
- 点击徽章查看详情弹窗

### 10.3 庆祝动画
- 徽章解锁时全屏弹出动画
- 徽章图标 + 名称 + "恭喜获得" 文字
- 3 秒后自动消失 或 点击关闭

## 11. 数据结构要点

```typescript
UserAchievement {
  id: string
  userId: string
  achievementId: string
  progress: number             // 当前进度
  target: number               // 目标值
  unlockedAt?: datetime        // 解锁时间
}

Achievement {
  id: string
  name: string
  icon: string
  description: string
  type: 'one_time' | 'milestone' | 'collaborative' | 'ultimate'
  target: number
  category: 'medication' | 'recording' | 'control' | 'team' | 'learning'
}

Streak {
  userId: string
  type: 'recording' | 'medication' | 'measurement'
  currentStreak: number        // 当前连续天数
  longestStreak: number        // 最长连续天数
  lastActiveDate: date
}

FamilyChallenge {
  id: string
  familyId: string
  type: string
  title: string
  description: string
  targetValue: number
  currentValue: number
  participantIds: string[]
  startDate: date
  endDate: date
  reward: string
  status: 'active' | 'completed' | 'expired'
}

OnboardingProgress {
  userId: string
  steps: {
    profile: boolean
    family: boolean
    medication: boolean
    firstRecord: boolean
    goal: boolean
  }
  completed: boolean
}
```

## 13. 异常处理

| 场景 | 处理 |
|------|------|
| Onboarding 中途退出 | 保存进度，下次登录继续 |
| 徽章解锁条件重复触发 | 仅首次解锁弹窗，后续不重复 |
| 打卡中断 | 不显示负面信息，鼓励重新开始 |
| 家庭挑战中成员退出 | 重新计算目标，可调整参与人 |

## 14. 验收标准

- [ ] 子女 Onboarding 流程完整（6步）
- [ ] 年轻患者 Onboarding 流程完整（7步）

- [ ] Onboarding 退出后进度保存
- [ ] 成就徽章列表正确展示已获得/未获得
- [ ] 到达条件时徽章自动解锁 + 庆祝动画
- [ ] 连续打卡天数正确统计
- [ ] 家庭挑战创建和进度展示正常
- [ ] 挑战达成触发庆祝
- [ ] 每周一早 8:00 推送周报
- [ ] 推送语气使用"关心"而非"警告"
- [ ] 第一周新用户引导覆盖核心操作
