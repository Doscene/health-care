# 06 饮食助手需求

## 1. 功能概述

双病兼容食谱推荐、食材替换建议、拍照记录饮食、买菜清单。核心差异化：**一道菜同时满足高血压（低钠）和糖尿病（低GI）需求**，两人份不用分锅做。

---

## 2. 用户故事

| 角色 | 场景 | 期望结果 |
|------|------|----------|
| 退休夫妻 | 每天发愁做什么饭，两个人病不一样不知道怎么吃 | 输入高血压+糖尿病，推荐适合两个人的一锅菜 |
| 老人 | 想吃红烧肉但不敢做 | App 告诉怎么做健康的红烧肉 |
| 年轻患者 | 中午外卖不知道能不能吃 | 拍照识别 AI 评级这顿安全/注意/超标 |
| 配偶 | 想帮患者做好吃的又不影响病情 | 获得替代食材建议和调味改进方案 |

---

## 3. 双病食谱推荐

### 3.1 推荐算法
- 输入：用户病种组合（高血压 / 糖尿病 / 双病）
- 筛选条件：低钠（每100g 钠 < 120mg）+ 低GI（GI < 55）
- 适配人数：1人份 / 2人份
- 推荐来源：内置食谱库（MVP 阶段预置 50+ 道菜）

### 3.2 食谱详情卡片
展示字段：
- 菜品名称 + 双病友好标签
- 菜品图片
- 营养标签：钠含量 | GI值 | 热量
- 食材替换建议："不要浇蒸鱼豉油，用葱姜蒜+少许柠檬汁提鲜"
- "加入今日菜单"按钮

### 3.3 搜索
- 按菜名搜索
- 按食材搜索
- 按餐类筛选：早餐/午餐/晚餐/加餐

---

## 4. 食材替换

### 4.1 功能描述
用户输入想吃的菜品 → 系统给出健康替代方案

### 4.2 替换规则（内置规则表）

| 想吃的 | 问题 | 替换建议 |
|--------|------|----------|
| 红烧肉 | 高油高糖 | 代糖替代冰糖，加魔芋块吸油，总量控制在200g内 |
| 白米饭 | 高GI主食 | 替换为杂粮饭（糙米+燕麦+黑米），GI降低40% |
| 炒菜 | 用盐多 | 起锅前放盐减少50%用量，用葱姜蒜辣椒提味 |
| 甜点 | 高糖 | 用代糖或少量水果替代 |
| 火锅 | 高钠高脂 | 清汤锅底，蘸料少麻酱多醋 |

### 4.3 食材替换交互
- 列表页每个菜品卡片右箭头
- 点击展开替换建议（折叠/展开）
- 替换建议以温馨提示卡片展示

---

## 5. 饮食记录

### 5.1 拍照记录
- 调用相机拍食物照片
- **MVP 阶段**：拍照片存储，不做 AI 自动分析
- 后续迭代接入 AI 估算碳水和盐分

### 5.2 快速记录（同04章）
- 从饮食页入口 → 跳转快速记录页 → 选择饮食 Tab
- 拍照 + 保存 → 关联饮食记录

### 5.3 今日菜单
- 从食谱详情"加入今日菜单"
- 今日菜单列表：早餐/午餐/晚餐分栏
- 支持删除已添加的菜品

---

## 6. 买菜清单

### 6.1 自动生成
- 基于今日菜单 + 食谱食材，自动汇总买菜清单
- 按食材类别分组：蔬菜/肉类/主食/调味品
- 勾选已购买

### 6.2 手动添加
- 输入框 + 添加按钮
- 支持删除单项
- 清空清单

---

## 7. 页面/交互说明

### 7.1 饮食页
- 顶部标题"饮食助手"，副标题"双病兼容 · 低钠低GI"
- **今日推荐**模块：
  - 横滑食谱卡片（含图片+菜名+标签+简略营养值）
  - 每张卡片 → 点击展开详情
- **想做这个？**模块（食材替换）：
  - 常见高油高糖菜品列表
  - 每项：emoji + 菜名 + 问题说明 + 折叠替换建议
- **快速记录**按钮：
  - 虚线方框 + 相机图标 + "拍照记录今日饮食"

### 7.2 食谱详情
- 大图区域
- 菜名 + "双病友好" / "低钠" / "低GI"标签
- 营养数据胶囊组
- 替换建议黄色背景卡片
- "加入今日菜单"按钮

### 7.3 今日菜单
- 从饮食页入口
- 早/午/晚三段
- 每段可添加/移除菜品
- 底部"生成买菜清单"按钮

## 8. 数据结构要点

```typescript
Recipe {
  id: string
  name: string
  image: string
  sodium: number            // 钠 mg/100g
  glycemicIndex: string     // "低" / "中" / "高"
  calories: number          // kcal
  suitableFor: ('hypertension' | 'diabetes')[]
  servings: number
  ingredients: string[]
  steps: string[]
  substitutionTips: string[]
}

SubstitutionRule {
  id: string
  originalFood: string
  problem: string
  suggestion: string
  tags: string[]
}

DietRecord {
  id: string
  userId: string
  mealType: 'breakfast' | 'lunch' | 'dinner' | 'snack'
  imageUrl?: string
  description?: string
  recordedAt: datetime
}

ShoppingList {
  id: string
  familyId?: string
  userId: string
  items: ShoppingItem[]
  createdAt: datetime
}

ShoppingItem {
  name: string
  category: 'vegetable' | 'meat' | 'staple' | 'condiment' | 'other'
  checked: boolean
}
```

## 9. 接口需求概要

| 接口 | 方法 | 路径 | 说明 |
|------|------|------|------|
| 食谱推荐 | GET | /api/recipes/recommend | query: diseases, servings |
| 食谱搜索 | GET | /api/recipes/search | query: keyword, mealType |
| 食谱详情 | GET | /api/recipes/:id | |
| 食材替换查询 | GET | /api/recipes/substitutions | query: foodName |
| 保存饮食记录 | POST | /api/diet-records | body: DietRecord |
| 获取饮食记录 | GET | /api/diet-records | query: userId, date |
| 获取/更新买菜清单 | GET/PUT | /api/shopping-list | |
| 加入今日菜单 | POST | /api/daily-menu | body: { recipeId, mealType } |
| 获取今日菜单 | GET | /api/daily-menu | query: userId, date |

## 10. 异常处理

| 场景 | 处理 |
|------|------|
| 食谱加载失败 | 展示占位卡片 + "加载失败，点击重试" |
| 搜索无结果 | 空状态："没找到相关食谱，试试其他关键词" |
| 拍照权限未授权 | 提示"请在设置中开启相机权限" |
| 拍照保存失败 | Toast + 本地暂存图片 |
