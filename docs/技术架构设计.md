### 3.3 数据可见性守卫 (VisibilityGuard) ⭐

这是整个隐私权限体系的核心实现：

```typescript
// visibility.guard.ts
@Injectable()
export class VisibilityGuard implements CanActivate {
  constructor(private prisma: PrismaService) {}

  async canActivate(context: ExecutionContext): Promise<boolean> {
    const request = context.switchToHttp().getRequest();
    const viewer = request.user;        // 当前登录用户
    const targetUserId = request.params.userId || request.query.userId;

    if (!targetUserId || viewer.id === targetUserId) return true; // 看自己

    // 查询两者在家庭圈中的关系 + 隐私设置
    const membership = await this.prisma.familyMember.findFirst({
      where: {
        family: { members: { some: { userId: viewer.id } } },
        userId: targetUserId,
      },
      select: { visibility: true, role: true },
    });

    if (!membership) return false; // 不在同一家庭圈

    // 根据请求的数据类型判断可见性
    const dataType = this.getDataType(request.path);
    return this.checkVisibility(membership.visibility, viewer.selfRole, dataType);
  }

  private checkVisibility(visibility: any, viewerRole: string, dataType: string): boolean {
    const setting = visibility[dataType];
    if (setting === 'all') return true;
    if (setting === 'spouse_only' && viewerRole === 'spouse') return true;
    if (setting === 'child_only' && viewerRole === 'child') return true;
    return false;
  }
}
```

## 四、多厂商OCR/语音兼容设计

### 4.1 设计目标

- 统一接口（Strategy Pattern），运行时按配置切换厂商
- 新增厂商只需实现接口，不修改调用方
- 主备自动 fallback：主厂商失败/低置信度时自动切换备选

### 4.2 接口定义

```typescript
// ocr-provider.interface.ts
export interface OcrResult {
  success: boolean;
  text: string;
  confidence: number;       // 0-1
  provider: string;         // 'baidu' | 'tencent' | 'alibaba'
}

export interface OcrProvider {
  readonly name: string;
  recognize(imageBase64: string, options?: OcrOptions): Promise<OcrResult>;
  isAvailable(): Promise<boolean>;
}

export interface OcrOptions {
  type?: 'general' | 'medicine_box' | 'bp_monitor';
}
```

### 4.3 门面服务 + Fallback 机制

```typescript
// ocr.service.ts
@Injectable()
export class OcrService {
  private providers = new Map<string, OcrProvider>();

  constructor(
    private config: OcrConfigService,
    baidu: BaiduOcrProvider,
    tencent: TencentOcrProvider,
    alibaba: AlibabaOcrProvider,
  ) {
    this.providers.set('baidu', baidu);
    this.providers.set('tencent', tencent);
    this.providers.set('alibaba', alibaba);
  }

  async recognize(imageBase64: string, options?: OcrOptions): Promise<OcrResult> {
    // 1. 尝试主厂商
    const primary = this.providers.get(this.config.primary);
    const result = await primary.recognize(imageBase64, options);
    if (result.success && result.confidence >= 0.6) return result;

    // 2. 按优先级 fallback
    for (const name of this.config.fallbacks) {
      try {
        const fb = await this.providers.get(name).recognize(imageBase64, options);
        if (fb.success) return fb;
      } catch { continue; }
    }

    return { success: false, text: '', confidence: 0, provider: 'none' };
  }
}
```

| `FamilyMember` | `(familyId, userId)` UNIQUE | 防重复 + 查询 |
| `BloodPressureRecord` | `(userId, recordedAt)` | 趋势/周报聚合 |
| `BloodSugarRecord` | `(userId, recordedAt)` | 血糖趋势 |
| `MedicationRecord` | `(userId, scheduledTime)` | 服药日历 |
| `RiskAlert` | `(userId, level, status)` | 活跃告警 |

### 5.3 MySQL 8.0 特性利用

| 需求 | MySQL 8.0 方案 |
|------|---------------|
| 趋势分析 | `ROW_NUMBER() OVER`, `LAG()` 窗口函数 |
| JSON半结构化 | `JSON` 类型 + `->>` 操作符（替代PG的JSONB） |
| 枚举 | `VARCHAR` + 应用层约束（比ENUM易扩展） |
| 聚合 | 应用层或多次查询（MySQL无Cube/Rollup的简洁语法） |

## 六、部署架构

```
┌────────────────────────────────────────────┐
│                Docker Compose                │
│                                              │
│  ┌──────────────┐   ┌──────────────────┐    │
│  │ Nginx :80/443│   │ NestJS :3000 x3  │    │
│  │ 反向代理+静态  │   │  (PM2 cluster)   │    │
│  └──────┬───────┘   └────────┬─────────┘    │
│         │                     │               │
│  ┌──────┴─────────────────────┴─────────┐    │
│  │           MySQL 8.0 :3306            │    │
│  │        (主库 + 每日dump备份)          │    │
│  └──────────────────────────────────────┘    │
│                                              │
│  ┌──────────────┐   ┌──────────────────┐    │
│  │ Redis :6379  │   │ MinIO :9000      │    │
│  │ 缓存+Bull队列 │   │  文件存储(OSS备选)│    │
│  └──────────────┘   └──────────────────┘    │
└────────────────────────────────────────────┘
```

- Nginx 路由：`/api/*` → NestJS:3000，`/static/*` → MinIO:9000
- NestJS 通过 PM2 cluster 模式多实例，Bull Queue 保证任务单次执行
- MySQL 每日凌晨自动 `mysqldump` + 保留最近7天备份
