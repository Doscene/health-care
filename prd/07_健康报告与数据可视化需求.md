# 07 健康报告与数据可视化需求

## 1. 功能概述

周报（子女端/家庭端）、月报、季度故事报告（年轻患者）、趋势图表、复诊数据导出。核心设计：子女被动接收而非主动监控，报告以温和图文形式呈现。

---

## 2. 用户故事

| 角色 | 场景 | 期望结果 |
|------|------|----------|
| 子女 | 每周一收到爸妈上周健康简报 | 一眼看到父母血压稳不稳、药有没有按时吃、有没有异常需要关心 |
| 年轻患者 | 想看自己血压这半年是好转还是恶化 | 季度趋势报告"故事化"呈现，如"这个夏天比春天下降了8mmHg" |
| 老人 | 去医院复诊，医生问近期数据 | 子女帮导出一张汇总表，血压血糖趋势一目了然 |
| 退休夫妻 | 月底看看这个月两人控制得怎么样 | 月报展示两人趋势对比，下个月设新目标 |

---

## 3. 周报（子女端）

### 3.1 报告内容
- **标题**："上周健康简报"
- **时间范围**：上周一至周日
- **按成员分卡片展示**：

#### 爸爸 · 张建国（高血压）
| 维度 | 展示 |
|------|------|
| 血压趋势 | "血压控制平稳，平均 132/84 mmHg，均在目标范围内" |
| 服药依从 | "7天全部按时服药，无漏服" |
| 测量习惯 | "血压测量 6 次，漏测 1 次（周四）" |
| 总体评价 | 😊👍 "爸爸本周表现很棒，继续保持！" |

#### 妈妈 · 王秀英（糖尿病）
| 维度 | 展示 |
|------|------|
| 血糖趋势 | "血糖偏高需关注，平均空腹 7.2 mmol/L，有 2 天超出 7.0" |
| 服药依从 | "二甲双胍按时服用，无漏服" |
| 异常提醒 | "妈妈近3天空腹血糖偏高，可能和饮食有关，建议电话问问" |
| 总体评价 | 🤔 "需要关注" |

### 3.2 下周提醒
- 复诊提醒："妈妈 5月15日（周四）内分泌科复诊"
- 检查提醒："爸爸 5月18日（周日）社区医院测血压"

### 3.3 分享功能
- "分享给家人"按钮 → 生成图文卡片 → 分享到微信
- 采用"关心"而非"警报"的语气

## 4. 家庭月报

### 4.1 报告内容
- 标题"家庭健康月报"
- 每月1日自动生成上月报告
- 内容包括：
  - 两人血压/血糖趋势对比图
  - 各自服药依从性柱状图
  - 本月达标天数统计
  - 复诊记录汇总
  - 家庭目标完成情况
  - 下月建议目标

### 4.2 趋势对比图
- 双折线图（两人收缩压叠在一起）
- X 轴：每周（4 周）
- Y 轴：血压值，标注目标线（140/90）
- 下方附文字小结

## 5. 季度故事报告（年轻患者专属）

### 5.1 核心设计理念
**叙事化呈现数据**——不是罗列数字，而是讲"你的健康故事"

### 5.2 报告结构
```
📖 李想的夏季健康故事（6月—8月）
├── 📊 血压：平均 128/84，比春季下降 8mmHg
│   └── 💬 "这个夏天你控制了 12 次应酬中的 8 次，表现不错"
├── 🍬 血糖：糖化血红蛋白从 7.2% 降至 6.8%
│   └── 💬 "接近目标值 6.5%，继续保持"
├── 💊 依从性：整体服药依从 96%，有 3 次漏服
│   └── 💬 "漏服主要在周末，试试设置周末专属提醒？"
├── 🔮 10年心血管风险：从 8.2% 降至 7.5%
│   └── 💬 "风险下降，继续保持健康习惯可以进一步降低"
└── 🎯 下季度目标建议：平均收缩压 ≤ 125
```

### 5.3 分享支持
- 一键生成匿名海报（去隐私）→ 分享到朋友圈
- 强化成就感和掌控感

## 6. 复诊数据导出

### 6.1 导出发起
- 复诊前3天自动提醒 + 数据导出入口
- 位置："我的" → 数据导出

### 6.2 导出内容
- 近期血压/血糖记录列表（默认近2周，可调范围）
- 统计摘要：平均值、最高值、最低值、异常次数
- 服药记录：依从性百分比、漏服明细
- 排版适配 A4 纸打印

### 6.3 导出格式
- PDF（推荐，可直接打印）
- 图片（方便微信发）

---

## 7. 趋势图表组件

### 7.1 血压趋势图
- 折线图：收缩压（蓝色折线）+ 舒张压（橙色折线）
- 背景：绿色正常区域、黄色偏高区域、红色危险区域
- 时间粒度：日/周/月可切换

### 7.2 血糖趋势图
- 折线图或散点图：按测量类型着色（空腹=黄色、餐后=红色）
- 标注目标线（空腹 < 7.0，餐后 < 10.0）

### 7.3 服药依从性图
- 柱状图：每天按时/延迟/漏服
- 月视图最常用

---

## 8. 页面/交互说明

### 8.1 周报页
- 顶部：标题 + 时间范围
- 成员卡片：头像 + 姓名 + 表情评价 + 详细指标
- 异常高亮（如有）
- 下周提醒深色卡片
- 底部分享按钮

### 8.2 数据导出页
- 选择导出成员（默认本人）
- 选择时间范围（快捷：近2周/近1月/近3月/自定义）
- 预览简要数据
- "生成报告" → 下载或分享

## 9. 数据结构要点

```typescript
WeeklyReport {
  id: string
  familyId: string
  weekStart: date
  weekEnd: date
  members: MemberWeeklySummary[]
  upcomingEvents: UpcomingEvent[]
  generatedAt: datetime
}

MemberWeeklySummary {
  userId: string
  name: string
  disease: string
  moodEmoji: string              // 😊 🤔 😟
  bpAvg?: { systolic: number, diastolic: number }
  bgAvg?: number
  bpStatus: 'normal' | 'elevated' | 'high'
  bgStatus: 'normal' | 'high'
  adherenceRate: number          // 服药依从性
  measurementCount: number       // 测量次数
  missedCount: number            // 漏测次数
  abnormalDays: number
  summaryText: string
  suggestion?: string
}

QuarterlyStoryReport {
  userId: string
  quarter: string
  bpTrend: { change: number, story: string }
  bgTrend: { change: number, story: string }
  adherence: { rate: number, missedDetails: string }
  cardiovascularRisk: { current: number, previous: number }
  nextGoal: string
}

UpcomingEvent {
  type: 'appointment' | 'checkup'
  memberId: string
  memberName: string
  title: string
  date: date
}
```

## 10. 接口需求概要

| 接口 | 方法 | 路径 | 说明 |
|------|------|------|------|
| 生成周报 | GET | /api/reports/weekly | query: familyId, weekStart |
| 生成月报 | GET | /api/reports/monthly | query: familyId, month |
| 季度故事报告 | GET | /api/reports/quarterly-story | query: userId, quarter |
| 复诊数据导出 | GET | /api/reports/export | query: userId, startDate, endDate, format |
| 趋势数据 | GET | /api/metrics/trend | query: userId, type, period, granularity |
| 复诊提醒列表 | GET | /api/appointments/upcoming | query: userId |

---

## 11. 异常处理

| 场景 | 处理 |
|------|------|
| 该周无数据 | 周报显示"本周暂无记录数据" + 鼓励开始记录 |
| 报告生成超时 | 异步生成，通知"报告生成中，完成后推送通知" |
| 导出数据量过大 | 分页处理，最多导出 12 周数据 |
| 家庭成员无权限查看 | 周报仅展示授权可见成员的数据 |
