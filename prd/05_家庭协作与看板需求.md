# 05 家庭协作与看板需求

## 1. 功能概述

家庭成员健康数据概览、夫妻协作任务、家庭目标设定、互相提醒、隐私边界管理。三个角色看到和使用的功能不同：老人看简单的成员列表，夫妻有协作玩法，子女看远程简报。

---

## 2. 用户故事

| 角色 | 场景 | 期望结果 |
|------|------|----------|
| 老人 | 看看老伴今天测血糖没有 | 家庭页看到老伴的健康状态，知道她今天有没有完成任务 |
| 退休夫妻 | 互相提醒测血糖吃药，一起完成家庭健康目标 | 看到对方的完成状态，互相提醒可获得默契值，月底看家庭月报 |
| 年轻患者 | 和配偶共同管理，但不想被过度监控 | 设置隐私边界，工作日数据仅自己可见，周末共享 |
| 子女 | 每周看一下父母的健康概览 | 家庭页看到父母两人的健康摘要卡片 |

---

## 3. 家庭成员列表

### 3.1 列表展示
- 家庭名称 + 成员总数（顶部）
- 成员卡片（三行分别展示不同类型成员）：

**患者本人卡片**（老人看到自己）：
- 头像 + 姓名 + "（我）"
- 病种 + 本周完成率（如"本周完成率 98%"）
- 状态标签：🟢 正常 / 🟡 关注 / 🔴 异常
- 子数据：平均血压/血糖 + 本周服药天数

**家庭成员卡片**（看别人的）：
- 头像 + 姓名 + 关系（配偶/子女/其他）
- 病种 + 角色
- 状态标签
- 子数据：近期平均指标 + 本周测量天数

**子女/照护者卡片**（不患病，仅显示角色）：
- 头像 + 姓名 + 关系
- "远程关怀"标签
- 无健康数据

### 3.2 切换查看对象
- 点击成员卡片 → 进入该成员的数据详情页
- 数据详情页展示该成员的指标趋势和服药情况
- 仅展示该成员授权可见的数据

## 4. 夫妻协作（年轻夫妻/退休夫妻）

### 4.1 协作模式
| 模式 | 适用对象 | 特点 |
|------|---------|------|
| 提醒-确认模式 | 退休夫妻 | 一方提醒另一方，对方确认后双方可见 |
| 共同目标模式 | 年轻夫妻 | 设定共同健康目标，双方数据透明 |

### 4.2 互相提醒
- 妻子提醒丈夫测血压 → 丈夫收到推送 → 丈夫完成记录
- 提醒记录计入"默契值"
- 老人端提醒入口：大按钮"提醒TA测血压/血糖"

### 4.3 默契值（轻游戏化）
- 夫妻默契值 = 互相提醒次数 × 1 + 共同完成目标数 × 5
- 家庭页展示默契值分数 + 星级图标
- 不设排行榜，仅内部可见

---

## 5. 家庭目标

### 5.1 目标设定
- 发起人创建目标，选择参与成员
- 目标类型：
  - 血压控制目标（如"两人本月血压达标天数 ≥ 20 天"）
  - 测量打卡目标（如"本月每人测血糖 ≥ 25 次"）
  - 运动目标（如"本周全家合计走满 50 公里"）
  - 自定义目标

### 5.2 目标进度
- 进度条 + 当前数值/目标数值
- 达标奖励：解锁家庭徽章
- 目标卡片示例：
  ```
  两人血压达标天数 >= 20 天
  ████████████░░░░░░░░ 65% 已完成 13 天，还差 7 天
  解锁「健康合伙人」徽章
  ```

## 6. 本周简报（子女/配偶端）

### 6.1 子女端简报
- 位置：家庭页内嵌或独立入口
- 内容：每个父母各自卡片
  - 血压/血糖指标概览
  - 服药依从百分比
  - 异常提醒（如有）
  - 简单评价（👍 表现很棒 / 🤔 需要关注）

### 6.2 配偶端简报
- 双方本周数据对比卡片
- 血压/血糖趋势小图
- 服药完成率

---

## 7. 隐私边界设置

### 7.1 年轻患者专属
- 入口：家庭页 → 隐私设置（或"我的"页进入）
- **工作日数据仅自己可见**：开启后，周一至周五的指标数据和应酬记录不对配偶同步
- **应酬记录不实时同步**：应酬相关记录仅在异常时触发共享
- **血糖波动细节控制**：可选择只共享"达标/不达标"标签而非具体数值

### 7.2 老人隐私设置
- 可在"我的"页设置：血压/血糖对子女可见、体重仅自己可见
- 老人端隐私设置入口更隐蔽，避免误操作

---

## 8. 页面/交互说明

### 8.1 家庭页
- 顶部：家庭名称 + 成员数
- 成员卡片列表
- 共同目标卡片（仅夫妻模式）
- 夫妻默契值卡片（仅夫妻模式）
- 本周简报入口
- 底部：邀请成员按钮

### 8.2 成员数据详情页
- 顶部：成员信息（头像+姓名+病种）
- 切换 Tab：指标趋势 / 服药记录 / 饮食记录
- 仅展示授权可见的数据
- 未授权数据显示"该成员未共享此数据"

## 9. 数据结构要点

```typescript
FamilyGoal {
  id: string
  familyId: string
  type: 'bp_control' | 'measure_checkin' | 'exercise' | 'custom'
  title: string
  targetValue: number
  currentValue: number
  unit: string
  participantIds: string[]
  startDate: date
  endDate: date
  achieved: boolean
}

MutualReminder {
  id: string
  fromUserId: string
  toUserId: string
  type: 'bp' | 'bg' | 'medication'
  createdAt: datetime
  respondedAt?: datetime
}

SynergyScore {
  familyId: string
  coupleIds: string[]
  score: number
  reminderCount: number
  goalCompleteCount: number
  period: string   // 本周/本月
}
```

## 10. 接口需求概要

| 接口 | 方法 | 路径 | 说明 |
|------|------|------|------|
| 获取家庭成员列表 | GET | /api/families/:id/members | |
| 获取成员健康摘要 | GET | /api/families/:id/members/:userId/summary | |
| 创建家庭目标 | POST | /api/families/:id/goals | |
| 更新目标进度 | PUT | /api/families/:id/goals/:goalId | |
| 发送提醒 | POST | /api/families/:id/remind | body: { toUserId, type } |
| 获取默契值 | GET | /api/families/:id/synergy | |
| 获取本周简报 | GET | /api/families/:id/weekly-summary | |
| 更新隐私设置 | PUT | /api/users/:id/visibility | |

## 11. 异常处理

| 场景 | 处理 |
|------|------|
| 提醒对象未注册 | 提示"对方还未加入家庭圈" |
| 对方数据不可见 | 显示"该成员未共享此数据"，不显示具体数值 |
| 目标参与人退出家庭 | 重新计算目标进度，可调整参与人 |
| 默契值计算周期内无互动 | 显示"本周还没有互动，提醒TA开始记录吧" |
