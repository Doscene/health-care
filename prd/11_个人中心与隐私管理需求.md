# 11 个人中心与隐私管理需求

## 1. 功能概述

个人资料、用药表管理、复诊计划、紧急联系人、隐私权限精细设置、数据导出、人生阶段切换。不同角色看到的入口和选项不同。

---

## 2. 用户故事

| 角色 | 场景 | 期望结果 |
|------|------|----------|
| 老人 | 查看自己吃什么药，下次什么时候复诊 | 我的页面大字列出药表和复诊时间 |
| 年轻患者 | 设置工作时间不给配偶看血糖数据 | 精细隐私开关，工作日数据自己可见 |
| 子女 | 帮父母管理用药信息 | 切换到父母视图后可编辑用药和复诊 |

---

## 3. 个人资料

### 3.1 资料内容
| 字段 | 编辑权限 | 说明 |
|------|---------|------|
| 头像 | 可编辑 | 支持拍照/相册选择 |
| 姓名 | 可编辑 | |
| 手机号 | 仅查看 | 变更需验证 |
| 出生年份 | 可编辑 | 用于年龄计算 |
| 病种 | 可编辑 | 高血压/糖尿病/双病/无 |
| 所属家庭圈 | 仅查看 | 家庭名称 |

### 3.2 账号安全
- 修改手机号（需验证码）
- 注销账号（需二次确认 + 验证码）
- 数据清除确认

---

## 4. 我的用药表

### 4.1 入口
- "我的" → 我的用药表
- 复用 03 章的药品列表功能

### 4.2 展示
- 药品卡片列表：药品名 + 规格 + 时间 + 状态
- 顶部 "+" 添加药品
- 点击进入编辑
- 左滑操作：编辑/暂停/删除

---

## 5. 我的复诊计划

### 5.1 复诊列表
- 未来复诊日程卡：
  - 日期 + 科室 + 医院
  - 倒计时（如"还有 5 天"）
- 历史复诊记录
- "+" 添加复诊计划

### 5.2 添加复诊
- 科室（心内科/内分泌科/肾内科 等预置）
- 医院名称
- 日期时间
- 备注
- 设置提醒（提前1天/3天/7天）

### 5.3 复诊提醒
- 提前 3 天推送提醒
- 提醒中含近期数据摘要链接

---

## 6. 紧急联系人

### 6.1 入口
- "我的" → 紧急联系人
- 复用 08 章的联系人设置

### 6.2 列表
- 联系人卡片：姓名 + 关系 + 电话 + 优先级序号
- 拖拽排序
- 添加/编辑/删除

### 6.3 一键呼救入口
- 紧急联系人页面顶部置顶红色按钮"一键呼救"
- 点击 → 确认拨打 120

---

## 7. 隐私权限设置

### 7.1 年轻患者隐私设置
| 设置项 | 选项 | 默认值 |
|--------|------|--------|
| 工作日数据仅自己可见 | 开/关 | 开启 |
| 应酬记录不实时同步 | 开/关 | 开启 |
| 血糖值细节共享 | 完整数值/仅达标状态 | 完整数值 |
| 体重数据可见范围 | 所有人/配偶/仅自己 | 仅自己 |
| 饮食记录可见范围 | 所有人/配偶/仅自己 | 配偶 |

### 7.2 老人隐私设置
| 设置项 | 选项 | 默认值 |
|--------|------|--------|
| 血压对子女可见 | 开/关 | 开启 |
| 血糖对子女可见 | 开/关 | 开启 |
| 体重对子女可见 | 开/关 | 关闭（默认） |
| 饮食记录对子女可见 | 开/关 | 关闭（默认） |

### 7.3 隐私设置交互
- 列表项形式，右侧开关（Toggle Switch）
- 每项有简要说明文字
- 即时生效，无需保存按钮

---

## 8. 人生阶段切换（年轻患者专属）

### 8.1 入口
- "我的" → "人生阶段"分组
- 仅在年轻患者角色显示

### 8.2 当前阶段
- 当前激活阶段高亮 + 🟢 已开启标签
- 可选阶段：备孕模式 / 职场模式 / 运动模式 / 日常模式
- 遗传风险评估入口
- 点击切换 → 确认弹窗（提示目标值和规则变化）

---

## 9. 数据导出

### 9.1 入口
- "我的" → 数据导出

### 9.2 导出设置
- 选择导出成员（自己/可切换管理的家庭成员）
- 选择时间范围：近2周/近1月/近3月/自定义
- 选择格式：PDF / 图片
- 生成后下载或分享

---

## 10. 其他

### 10.1 设置项
- 通知设置（开启/关闭各类型通知）
- 语音提醒（仅老人端）
- 大字体模式开关（老人端默认开启）
- 清除缓存
- 关于我们 / 版本号
- 用户协议 / 隐私政策

### 10.2 照护者切换
- 如果用户同时是患者和照护者（如年轻患者也是父母的子女）
- "我的"页顶部角色切换入口
- 当前角色标签 + 点击切换

## 12. 数据结构要点

```typescript
Appointment {
  id: string
  userId: string
  department: string           // 科室
  hospital: string
  date: datetime
  notes?: string
  remindBefore: number        // 提前X天提醒
  status: 'upcoming' | 'completed' | 'cancelled'
  createdAt: datetime
}

AppSettings {
  userId: string
  notifications: {
    medicationReminder: boolean
    measurementReminder: boolean
    appointmentReminder: boolean
    weeklyReport: boolean
    riskAlert: boolean
  }
  voiceReminder: boolean        // 老人端
  elderlyFont: boolean          // 老人端大字体
}
```

## 13. 接口需求概要

| 接口 | 方法 | 路径 | 说明 |
|------|------|------|------|
| 获取用户资料 | GET | /api/users/:id/profile | |
| 更新用户资料 | PUT | /api/users/:id/profile | |
| 复诊计划 CRUD | GET/POST/PUT/DELETE | /api/appointments | |
| 隐私设置 | GET/PUT | /api/users/:id/visibility | |
| 应用设置 | GET/PUT | /api/users/:id/settings | |
| 切换角色视图 | POST | /api/users/switch-view | body: { targetUserId } |
| 注销账号 | POST | /api/users/:id/deactivate | |
| 数据导出 | GET | /api/export | query: userId, format, startDate, endDate |
